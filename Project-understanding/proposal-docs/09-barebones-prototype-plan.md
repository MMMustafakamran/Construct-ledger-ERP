# Barebones Accounting Prototype Plan

## Goal

Create the simplest believable accounting prototype the client can approve quickly.

## Prototype Screens

1. dashboard
2. chart of accounts
3. vendors list
4. customers list
5. vendor invoice form/list
6. customer invoice form/list
7. payments and receipts
8. cash and bank summary

## Demo Story

1. create a vendor
2. enter a vendor invoice
3. record a payment from bank
4. see payable reduce
5. create a customer invoice
6. record receipt from customer
7. see receivable reduce
8. see dashboard and bank balance update

## Recommended Build Order

1. accounts
2. vendors and customers
3. invoices
4. payments and receipts
5. cash/bank summary
6. dashboard

## Time-Saving Rule

Do not build advanced accounting features in version 1.

Only build what is needed to prove:

- invoice entry
- payment tracking
- cash visibility
- receivables and payables summary

## Expansion Path Later

After this MVP is approved, the next upgrades can be:

- job tags or simple job costing
- fixed assets
- depreciation
- payroll
- inventory
- detailed reports
